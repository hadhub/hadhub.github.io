🏁 Introduction⌗

Après avoir mis en place le lab et effectué un premier scan, nous allons voir les premières étapes pour passer administrateur du premier domaine.

🔍 Énumération - Scan nmap des 3 machines⌗

Premier scan sur l’ip 192.168.56.10 :

  • 53/tcp open domain -> Domain name server, assure la résolution des noms de domaine.

  • 88/tcp open kerberos-sec -> Kerberos, gestion des authentifications et des sessions via un système de ticket

  • 389/tcp open ldap -> Annuaire LDAP, stocke les objets (utilisateurs, ordinateurs, groupes etc…) AD du domaine

  • Domaine : sevenkingdoms.local

  • FQDN : kingslanding.sevenkingdoms.local

Second scan sur l’ip 192.168.56.11 :

  • 53/tcp open domain -> Domain name server, assure la résolution des noms de domaine.

  • 88/tcp open kerberos-sec -> Kerberos, gestion des authentifications et des sessions via un système de ticket

  • 389/tcp open ldap -> Annuaire LDAP, stocke les objets (utilisateurs, ordinateurs, groupes etc…) AD du domaine

  • Domaine : north.sevenkingdoms.local

  • FQDN : winterfell.north.sevenkingdoms.local

Troisième scan sur l’ip 192.168.56.22 :

  • 80/tcp open http -> Service Web

  • 1433/tcp open ms-sql-s -> Service MSSQL (Serveur SQL Microsoft)

  • 3389/tcp open ms-wbt-server -> Remote Desktop Protocol

  • FQDN : castelblack.north.sevenkingdoms.local

cf Scan complet

Ajout des informations suivantes dans /etc/hosts afin de pouvoir faire de la résolution de nom :

192.168.56.10   kingslanding.sevenkingdoms.local sevenkingdoms.local
192.168.56.11   winterfell.north.sevenkingdoms.local north.sevenkingdoms.local
192.168.56.22   castelblack.north.sevenkingdoms.local
